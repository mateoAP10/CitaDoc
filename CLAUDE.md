# CitaDoc Core Rules

## 1. Prioridad absoluta
Mantener CitaDoc: premium, rápido, mobile-first, elegante, consistente. No introducir complejidad innecesaria.

## 2. Antes de modificar
- Entender el flujo completo, revisar dependencias, validar impacto mobile, preservar backwards compatibility.
- NUNCA romper: billing, PayPhone, Supabase auth, disponibilidad médica, renderer web, AI generation flow.

## 3. IA y tokens — optimizar SIEMPRE
- Prompts compactos, evitar prompts gigantes, reutilizar contexto, usar fallbacks locales cuando no se necesite IA real.
- No llamar Kimi para tareas triviales/UI.
- Reservar IA para: storytelling, medical branding, SEO, patient experience, clinical intelligence.

## 4. UX
- Cada flujo: premium, claro, emocional, rápido.
- Evitar: modales rotos, iframes feos, loaders infinitos, flows duplicados, interfaces legacy.

## 5. Website Builder
- La IA NO debe solo cambiar texto. Debe generar: identidad, estructura, composición visual, tono, percepción premium.
- Siempre priorizar: layouts dinámicos, reveal wow, storytelling médico, diferenciación por especialidad.

## 6. Código
- Cambios pequeños, refactors limpios, eliminar legacy muerto, funciones reutilizables, evitar duplicación.
- Antes de entregar: validar sintaxis, revisar console errors, verificar mobile, probar end-to-end.

## 7. Estilo visual — Apple-level polish, luxury healthcare, calm tech, premium AI
- Preferir: glassmorphism sutil, spacing limpio, animaciones suaves, tipografía editorial, transiciones cinematográficas.
- Evitar: dashboards genéricos, colores agresivos, sombras pesadas, layouts sobrecargados.

## 8. Filosofía
No construir "otro software médico". Construir "la capa AI premium del médico moderno".
